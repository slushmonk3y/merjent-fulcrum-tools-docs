Development Guides

Variable Names

I have tried to use a naming process for functions and variable to make it easier to understand without getting too long. The majority of variables and functions are named using the following abbreviations. The first portion of a function name should be an abbreviation of the type of data it applies to. (ie: in the Helpers class any function that deals with processing an Array will follow the format of: arr....)
arr: Array
str: String
dt: Date
rows: Array of Objects(or JSONs)
iso: Date - as iso string
dn: data_name
dns: data_name(s) - Array of 'data_names' exc: Exclude - 'data_names' to not process, usually in an array
mk/make: Make - Method returns a JS type to use in other functions
pop: Populate - Populates fulcrum field, calls SETVALUE() at end
build: Build - Method returns html string, used in ReportHelpers rep: Repeatable - Method for fulcrum repeatables

Utils js.node

To install the class above in a node environment (ie. fulcrum data events) change the "var U = new Utils;" to:

    module.exports.U = new Utilss;

To use the utility in a fulcrum calculation expression, or other situations where it can not be installed as a module, leave the var U=new Utils line at the end.
