Python Tools - Batch Operations

Overview

The tools/ directory contains Python scripts for performing batch operations across all Fulcrum apps that use MTools. These tools enable you to update code, search/replace patterns, and perform administrative tasks efficiently across dozens of apps at once.

When to Use

Use Python tools when you need to:

  • Deploy updated MTools code to all apps
  • Bulk search/replace code patterns
  • Refactor function names across multiple apps
  • Export app configurations or data events
  • Perform administrative tasks at scale

What You Get

  • Centralized architecture - Reusable infrastructure for looping through apps
  • Update deployment - Push new MTools versions to all apps
  • Search/replace - Find and modify code across all apps
  • Export tools - Extract data events and configurations
  • Extensible framework - Easy to create new custom tools

Architecture

└── EXAMPLE_*.py                   # Example tools showing patterns

Core Modules

core/fulcrum_client.py

Centralized Fulcrum API configuration and client singleton.

from core.fulcrum_client import get_client, FULCRUM_TOKEN, MTOOLS_LIST_APP

fulcrum = get_client()  # Returns configured Fulcrum client

Configuration:

  • FULCRUM_TOKEN - Your API token
  • MTOOLS_LIST_APP - Form ID of the app that lists all MTools apps

core/app_looper.py

Infrastructure for looping through MTools apps and report templates.

from core.app_looper import loop_apps, loop_templates

# loop_apps handles:
# - Fetching records from MTools list app
# - Extracting form IDs and template names
# - Calling your handler for each app
# - Accumulating results

core/base_tool.py

Base class providing common structure for all tools.

from core.base_tool import BaseMToolTool

class MyTool(BaseMToolTool):
    def handle_app(self, form, template_names, context):
        # Process one app's data events
        pass

    def print_results(self, results, context):
        # Display results
        pass

Installation & Setup

Prerequisites

pip install fulcrum-python requests

Configuration

  1. Get your Fulcrum API token:

    • Log into Fulcrum
    • Go to Settings → API
    • Copy your API token

  2. Configure the Fulcrum client:

Edit tools/core/fulcrum_client.py:

# Your Fulcrum API token
FULCRUM_TOKEN = 'your_token_here'

# Form ID of your MTools app list
MTOOLS_LIST_APP = 'your_list_app_form_id'

Security Note: Consider using environment variables instead:

import os
FULCRUM_TOKEN = os.getenv('FULCRUM_API_TOKEN')

Using Existing Tools

updateMtools.py

Update MTools code across all apps from the latest dist files, with optional search/replace.

Basic Usage

# Update MTools code only
python updateMtools.py

What it does:

  1. Loads latest dist files from ../dist/
  2. Loops through all apps in MTools list
  3. Updates data events with merjent-tools-de.mmin.js or merjent-tools-mapp.mmin.js
  4. Updates report templates with merjent-tools-report.mmin.js
  5. Preserves custom code outside region markers

Search/Replace Options

Basic string replacement:

python updateMtools.py --search "oldFunction" --replace "newFunction"

Multiple replacements:

python updateMtools.py \
  --search "var " --replace "const " \
  --search "oldVar" --replace "newVar"

Regex replacement:

# Comment out all console.log statements
python updateMtools.py \
  --search "console\.log\(" --replace "// console.log(" \
  --regex

Target specific script types:

# Only apply to reports
python updateMtools.py \
  --search "REPORT_MODE" --replace "PDF_MODE" \
  --apply-to reports

# Only apply to data events
python updateMtools.py \
  --search "oldMethod" --replace "newMethod" \
  --apply-to data-events

Command-Line Arguments

  • --search PATTERN - Search pattern (can be repeated for multiple replacements)
  • --replace TEXT - Replacement text (pairs with --search, can be repeated)
  • --regex - Treat search patterns as regex instead of literal strings
  • --apply-to {all|data-events|reports} - Where to apply search/replace (default: all)

Processing Order

  1. MTools marker-based replacement happens first - Updates the MTools library code
  2. Search/replace operations applied second - Modifies entire script (both MTools region and custom code)

This allows you to update both the MTools library AND make bulk changes to your custom code in a single operation.

Common Use Cases

Update MTools and rename a function:

python updateMtools.py --search "calculateID" --replace "generateReportID"

Update MTools and modernize JavaScript:

python updateMtools.py \
  --search "var " --replace "const " \
  --search "function " --replace "const "

Update MTools and refactor error handling:

python updateMtools.py \
  --search "console\.log\('ERROR:" --replace "console.error('ERROR:" \
  --regex

findMtoolFunctions.py

Search for and optionally replace strings across all data events and report templates.

Search Only

# Find all uses of a function
python findMtoolFunctions.py -s "oldFunctionName"

Search and Replace

# Replace all occurrences
python findMtoolFunctions.py -s "oldName" -r "newName"

Multiple Search/Replace Pairs

python findMtoolFunctions.py -d '[{"old1":"new1"}, {"old2":"new2"}]'

Use Cases

  • Find where a function is used before refactoring
  • Rename variables across all apps
  • Update API endpoints
  • Fix typos in field names

Creating a New Tool

Creating a new batch operation tool is simple - just implement 3 methods:

Step 1: Create Tool Class

"""My new tool description"""
from core.base_tool import BaseMToolTool

class MyNewTool(BaseMToolTool):
    """Tool description shown in --help"""

    def setup_args(self, parser):
        """Add tool-specific CLI arguments"""
        parser.add_argument('-x', '--example', help='Example argument')

    def handle_app(self, form, template_names, context):
        """
        Process one app

        Args:
            form: Fulcrum form object
            template_names: List of report template names
            context: Dictionary with CLI args and iteration info

        Returns:
            List of result dictionaries
        """
        # Your logic here
        return [{'formName': form['form']['name'], 'result': 'success'}]

    def print_results(self, results, context):
        """Print results summary"""
        print(f"Processed {len(results)} apps")

if __name__ == '__main__':
    MyNewTool().run()

Step 2: Run Your Tool

python my_new_tool.py

That's it! The base class handles:

  • Argument parsing
  • Fetching records from MTools list
  • Looping through apps
  • Progress display
  • Calling your handlers

Advanced Tool Development

Context Dictionary

The context dictionary passed to handlers contains:

{
    # From CLI arguments
    'arg1': value1,
    'arg2': value2,

    # Added by loop_apps for each iteration
    'record_index': 0,           # Current record index
    'records_len': 10,           # Total number of records
    'record': {...},             # Full Fulcrum record from MTools list
    'list_fld_dic': {...}        # Field mappings for MTools list app
}

Processing Report Templates

If your tool needs to process report templates, override handle_report():

def handle_report(self, form_name, template, context):
    """
    Process one report template

    Args:
        form_name: Name of the form
        template: Report template data from Fulcrum API
        context: Dictionary with parameters

    Returns:
        List of result dictionaries
    """
    # Your logic here
    return []

Then call loop_templates() from handle_app():

from core.app_looper import loop_templates

def handle_app(self, form, template_names, context):
    results = []

    # ... process data events ...

    # Process report templates
    template_results = loop_templates(
        template_names,
        form,
        self.handle_report,
        context
    )
    if template_results:
        results.extend(template_results)

    return results

Example: Export Data Events

This example tool exports all data events to files:

"""Export data events from all MTools apps"""
import os
from core.base_tool import BaseMToolTool

class ExportDataEvents(BaseMToolTool):
    """Export all data events to local files"""

    def setup_args(self, parser):
        parser.add_argument(
            '-o', '--output-dir',
            default='./exports',
            help='Output directory for exported files'
        )

    def handle_app(self, form, template_names, context):
        output_dir = context.get('output_dir', './exports')

        # Create output directory if needed
        os.makedirs(output_dir, exist_ok=True)

        form_name = form['form']['name']
        script = form['form']['script']

        if script:
            # Clean filename
            filename = form_name.replace(' ', '_').replace('/', '_')
            filepath = os.path.join(output_dir, f'{filename}.js')

            # Write to file
            with open(filepath, 'w', encoding='utf-8') as f:
                f.write(script)

            return [{
                'formName': form_name,
                'exported': True,
                'filepath': filepath
            }]

        return [{
            'formName': form_name,
            'exported': False,
            'reason': 'No data events'
        }]

    def print_results(self, results, context):
        exported = [r for r in results if r.get('exported')]
        not_exported = [r for r in results if not r.get('exported')]

        print(f"\nExported {len(exported)} data events")
        print(f"Skipped {len(not_exported)} apps (no data events)")

        if exported:
            print(f"\nFiles saved to: {context.get('output_dir')}")

if __name__ == '__main__':
    ExportDataEvents().run()

Usage:

python export_data_events.py -o ./my_exports

Best Practices

1. Test on a Single App First

Before running on all apps, test your tool on one app:

# In handle_app()
form_name = form['form']['name']
if form_name != 'Test App':
    return []  # Skip all except Test App

# Your logic here

2. Always Provide Progress Feedback

def handle_app(self, form, template_names, context):
    form_name = form['form']['name']
    record_index = context.get('record_index', 0)
    records_len = context.get('records_len', 0)

    print(f'Processing {form_name} ({record_index+1}/{records_len})...')

    # Your logic

3. Handle Errors Gracefully

def handle_app(self, form, template_names, context):
    try:
        # Your logic
        return [{'formName': form_name, 'success': True}]
    except Exception as e:
        print(f'ERROR processing {form_name}: {e}')
        return [{'formName': form_name, 'success': False, 'error': str(e)}]

4. Validate Results

def print_results(self, results, context):
    successful = [r for r in results if r.get('success')]
    failed = [r for r in results if not r.get('success')]

    print(f"\nSuccessful: {len(successful)}")
    print(f"Failed: {len(failed)}")

    if failed:
        print("\nFailed apps:")
        for r in failed:
            print(f"  - {r['formName']}: {r.get('error', 'Unknown error')}")

5. Back Up Before Bulk Operations

Before running destructive operations (updates, deletions), back up your data:

# Export all data events first
python export_data_events.py -o ./backup_$(date +%Y%m%d)

# Then run your update
python updateMtools.py --search "old" --replace "new"

Troubleshooting

Issue: API token not working

Cause: Token not configured or expired

Solution:

# Check token is set
from core.fulcrum_client import FULCRUM_TOKEN
print(FULCRUM_TOKEN)  # Should not be None or empty

# Test API connection
fulcrum = get_client()
try:
    forms = fulcrum.forms.search()
    print(f"Connected! Found {len(forms['forms'])} forms")
except Exception as e:
    print(f"Connection failed: {e}")

Issue: MTools list app not found

Cause: MTOOLS_LIST_APP form ID is incorrect

Solution:

# List all forms to find the correct ID
fulcrum = get_client()
forms = fulcrum.forms.search()
for form in forms['forms']:
    print(f"{form['name']}: {form['id']}")

Issue: Tool runs but doesn't update apps

Cause: Not calling the Fulcrum API update method

Solution:

# Make sure you're actually updating
response = self.fulcrum.forms.update(app_id, form)

# Check response
if 'errors' in response.get('form', {}):
    print(f"Update failed: {response['form']['errors']}")

Benefits of This Architecture

  1. DRY - No duplicating Fulcrum client setup, looping logic, etc.
  2. Consistent - All tools have the same CLI pattern and structure
  3. Scalable - Adding a new tool is just implementing 3 methods
  4. Clear separation - Infrastructure vs tool-specific logic
  5. Easy to test - Can test individual methods in isolation
  6. Flexible - Can customize everything while keeping common patterns

Old Pattern (loopMtoolsApps.py)

import loopMtoolsApps as LMA

def handleApp(form, templateNames, replaceDict):
    # ... logic ...

results = LMA.loopApps(records, recordsLen, handleApp, replaceDict)

New Pattern (BaseMToolTool)

from core.base_tool import BaseMToolTool

class MyTool(BaseMToolTool):
    def handle_app(self, form, template_names, context):

if __name__ == '__main__':

  • Result printing
  • Progress display
